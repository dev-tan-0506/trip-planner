# Phase 07 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn |
| Preset | `radix-nova` in both `apps/web/components.json` and `packages/ui/components.json` |
| Component library | Radix UI primitives via shadcn |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge insets |
| sm | 8px | Compact element spacing, chip internals |
| md | 16px | Default element spacing, input padding, card internals |
| lg | 24px | Section padding, grouped card spacing |
| xl | 32px | Layout gaps, shell panel padding |
| 2xl | 48px | Major section breaks, hero internals |
| 3xl | 64px | Desktop page spacing, large shell offsets |

Exceptions: `44px` minimum touch target for icon-only actions and bottom-nav items; `80px` minimum floating action button size on desktop shell surfaces.

---

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.5 |
| Label | 14px | 800 | 1.3 |
| Heading | 28px | 800 | 1.2 |
| Display | 56px | 800 | 1.05 |

Use only these four size roles in Phase 07 primitives and shell blocks. Display is reserved for hero statements and top-level section headlines; labels stay uppercase or short-title only.

---

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#f9f6f5` | App background, page canvas, large resting surfaces |
| Secondary (30%) | `#f3f0ef` | Section surfaces, nav rails, search containers, grouped shells |
| Accent (10%) | `#a93000` | Primary CTAs, active navigation, focused quick actions, hero emphasis chips |
| Destructive | `#b31b25` | Delete, remove, urgent destructive confirmation, SOS escalation only |

Accent reserved for: primary CTA buttons, active shell nav items, hero highlight text, focused action pills, notification count dots, and selected high-priority status chips. Do not use accent for every link, every secondary button, or neutral card chrome.

Support semantic colors: `#5a3bdd` for discovery/proposal/poll emphasis, `#006a33` for success/completion states, `#ff784d` for primary-container fills, `#fb5151` for error-container fills.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | `Tạo chuyến đi` |
| Empty state heading | `Chưa có gì để quẩy hết` |
| Empty state body | `Tạo một mục mới hoặc chọn gợi ý có sẵn để bắt đầu nhanh.` |
| Error state | `Có lỗi xảy ra rồi. Thử lại một lần nữa hoặc tải lại trang nếu vẫn chưa ổn.` |
| Destructive confirmation | `Xóa mục`: `Xóa mục này hả? Hành động này không hoàn tác được đâu.` |

Copy tone contract: Vietnamese-first, playful, short, and scannable. Use casual energy without unreadable slang spam. Avoid mojibake entirely.

## Semantic Surface Contract

| Semantic tier | Token target | Contract |
|---------------|--------------|----------|
| App background | `surface` / `background` | Warm off-white full-page base; never pure white wall-to-wall |
| Section surface | `surface-container-low` | Groups content without borders; default shell grouping plane |
| Lifted card | `surface-container-lowest` | White or near-white card used for actionable content and data blocks |
| High surface | `surface-container-high` | Supports tab rails, grouped controls, and layered shell chrome |
| Glass overlay | translucent white over `surface` with `24px` blur | Used for floating nav, top bars, and overlay affordances |
| CTA emphasis | `primary` + `primary-container` | Gradient or paired solid/container treatment for high-priority actions |
| Urgency surface | `error-container` / destructive tint | Used as fill-based warning treatment; avoid red-outline-only design |

No-divider rule: do not separate sections with 1px rules by default. Prefer tonal shifts, spacing, shadow, and radius. If a separator is necessary for accessibility or dense data, use `outline-variant` at `15%` opacity.

## Radius, Elevation, and Motion

| Property | Contract |
|----------|----------|
| Radius | Default `16px`, large cards `32px`, hero and major shells `48px`, pills `9999px` |
| Shadow | Use soft ambient shadows only: blur `32px` to `64px`, opacity `6%` to `10%`, tinted from on-surface instead of pure black |
| Blur | Glass surfaces use `24px` backdrop blur; smaller floating chips can step down to `16px` |
| Motion | Use entry reveal, hover lift, and active press only; duration `150ms` fast feedback, `300ms` section reveal, `500ms` hero entrance |
| Hover | Cards may lift by `-4px` to `-8px`; buttons may scale to `0.98` on press and `1.02` on hover |

Motion must support clarity first. Disable decorative motion for loading, errors, or dense operational states where movement competes with task completion.

## Primitive Inventory

Phase 07 must deliver these reusable primitives in `packages/ui`:

| Primitive | Contract |
|-----------|----------|
| Button | Primary, secondary, ghost, destructive, icon-only; all variants share pill or extra-round geometry |
| Input | Text input, search input, and textarea-ready field wrapper with icon slot and error/help text |
| Card | Base surface card plus lifted-card variant with no-divider defaults |
| Badge | Status, category, and counter badges with semantic color variants |
| Dialog | Confirmation and modal primitive with glass-capable header/footer treatment |
| Tabs | Rounded segmented control for shell-level switching |
| Avatar | Single avatar, stacked avatar group, and status-dot support |
| IconButton | 44px min target, circular or rounded-2xl treatment only |
| Sheet | Mobile-first bottom sheet for composer and detail overflow patterns |
| Skeleton | Loading placeholders for cards, lists, and shell header content |
| EmptyState | Shared empty-state frame with icon/art slot, title, body, and primary action |
| ErrorState | Shared error-state frame with retry action and optional detail line |

Every primitive must support loading, empty, error, disabled, and responsive behavior where applicable. Disabled states use muted fill and reduced contrast, not opacity alone.

## Shared Shell Blocks

Phase 07 must establish these reusable shell blocks in `packages/ui` or a clearly owned shared-shell area:

| Block | Contract |
|-------|----------|
| AppHeader | Product mark, search field, notification trigger, account affordance, optional contextual action |
| SearchField | Rounded search input with icon-leading layout and quiet container surface |
| NotificationTrigger | Icon button with optional unread dot or count |
| ProfileChip | Avatar + short label + optional status affordance in rounded container |
| HeroPanel | Oversized heading, support text, CTA row, optional media/gradient backing |
| SectionFrame | Shared padded section wrapper for grouped cards without divider lines |
| ActionPillRow | Horizontal quick-action cluster for tabs, day selectors, and shell shortcuts |
| GlassNav | Floating glass navigation for desktop rail or mobile bottom navigation |
| StatusBanner | Fill-based success, warning, or destructive announcement panel |
| FloatingActionSurface | Desktop FAB and mobile primary quick action shell with strong radius and shadow |

These blocks exist to stop route-level reinvention. Feature phases may compose them, but must not rebuild header, search, or nav chrome ad hoc.

## Interaction State Contract

| State | Contract |
|-------|----------|
| Loading | Use skeletons or progress affordances inside the target surface; avoid layout shift |
| Empty | Use friendly headline, one-line explanation, and one clear next action |
| Error | Use fill-based error container plus retry path; keep message human-readable |
| Disabled | Preserve readability, reduce emphasis, and block hover-only affordances |
| Active/Selected | Show through fill, accent, or depth change; do not rely on text color only |
| Focus | Visible keyboard ring using accent tint on every interactive primitive |

## Responsive Contract

Desktop-first, mobile-safe means:

| Breakpoint behavior | Contract |
|---------------------|----------|
| Desktop | Shell may use side rail, wide hero compositions, multi-column cards |
| Tablet | Collapse secondary chrome before collapsing core content; preserve search, notifications, and primary CTA access |
| Mobile | Swap rail for bottom nav or sheet; maintain 44px targets, stack hero/actions, keep quick actions horizontally scrollable rather than shrinking to unreadable pills |

Never hide critical trip actions behind hover. Mobile layouts must retain search, notifications, profile access, and the current screen's primary action without requiring a desktop-only pattern.

## Implementation Constraints

| Area | Contract |
|------|----------|
| Token ownership | Move token authority out of `apps/web/app/globals.css` into shared `packages/ui` styling sources consumed by web |
| Import paths | Generated or hand-written component imports must resolve through `@repo/ui/components` and `@repo/ui/lib/utils` |
| Alias safety | Reject any generated path under `apps/web/@repo/*`; the alias-resolution quirk is an implementation issue to fix, not an allowed contract |
| App globals | `apps/web/app/globals.css` should become app wiring and overrides only, not the long-term home of the design language |
| Feature routes | Routes may compose shell blocks and primitives, but should not define net-new token values locally |
