# Plan 01-01: Infrastructure & Authentication Foundation

## Objective
Establish the Turborepo monorepo with Next.js (Web), NestJS (API), and Prisma. Set up base authentication using Supabase.

## Tasks

<task>
<read_first>
- `.planning/phases/01-foundation-onboarding/01-CONTEXT.md`
</read_first>
<action>
Initialize a Turborepo at the project root with the name "minh-di-dau-the".
Use the `npx create-turbo@latest .` command and select npm.
Strip out default apps and initialize:
1. `apps/web`: Next.js 14+ (App Router) with Tailwind CSS.
2. `apps/api`: NestJS application (`npx @nestjs/cli new api`).
</action>
<acceptance_criteria>
- `package.json` contains `turbo` in devDependencies.
- `apps/web/package.json` and `apps/api/package.json` exist.
</acceptance_criteria>
</task>

<task>
<read_first>
- `apps/api/package.json`
</read_first>
<action>
Create `packages/database` directory and initialize Prisma.
1. `npm install prisma --save-dev`
2. `npx prisma init`
3. Define the `User` schema in `schema.prisma` with `id`, `email`, `name`, `avatarUrl`, `healthProfile`.
</action>
<acceptance_criteria>
- `packages/database/prisma/schema.prisma` exists and contains `model User`.
</acceptance_criteria>
</task>

<task>
<read_first>
- `apps/web/package.json`
</read_first>
<action>
Install `@supabase/supabase-js`.
Create a basic Supabase client instance in `apps/web/src/lib/supabase.ts`.
Create an Auth Context provider in `apps/web` to handle Magic Link and Social Login UI.
</action>
<acceptance_criteria>
- `apps/web/src/lib/supabase.ts` exists and exports a loaded supabase client.
- Auth login page handles Google OAuth and Magic Link.
</acceptance_criteria>
</task>

## Verification
- Run `npm run test` in `apps/api` to ensure NestJS boots.
- Run `npm run lint` across the monorepo via Turborepo.

## Must Haves
- Monorepo structure successfully orchestrated by Turborepo.
- Prisma schema for User successfully validates.
