# Plan 01-01: Infrastructure & Authentication Foundation

## Objective
Establish the Turborepo monorepo with Next.js (Web), NestJS (API), and Prisma. Set up base authentication using self-built Passport.js + JWT (replacing Supabase).

## Tasks

<task>
<read_first>
- `.planning/phases/01-foundation-onboarding/01-CONTEXT.md`
</read_first>
<action>
Initialize a Turborepo at the project root with the name "minh-di-dau-the".
Use the `npx create-turbo@latest .` command and select npm.
Strip out default apps and initialize:
1. `apps/web`: Next.js 14+ (App Router) with Tailwind CSS.
2. `apps/api`: NestJS application (`npx @nestjs/cli new api`).
</action>
<acceptance_criteria>
- `package.json` contains `turbo` in devDependencies.
- `apps/web/package.json` and `apps/api/package.json` exist.
</acceptance_criteria>
</task>

<task>
<read_first>
- `apps/api/package.json`
</read_first>
<action>
Create `packages/database` directory and initialize Prisma.
1. `npm install prisma --save-dev`
2. `npx prisma init`
3. Define the `User` schema in `schema.prisma` with `id`, `email`, `name`, `avatarUrl`, `healthProfile`.
</action>
<acceptance_criteria>
- `packages/database/prisma/schema.prisma` exists and contains `model User`.
</acceptance_criteria>
</task>

<task>
<read_first>
- `apps/web/src/lib/api-client.ts`
</read_first>
<action>
Implement the Authentication UI in `apps/web`:
1. Create a React Auth Context Provider (`AuthContext.tsx`) to manage user state globally using Zustand or React Context.
2. Build the Registration and Login UI pages mapping to the API (`/auth/register` and `/auth/login`).
3. Ensure UI supports fallback paths and handles Token auto-refresh seamlessly.
</action>
<acceptance_criteria>
- Auth Context provider exists.
- Login and Register forms exist and successfully authenticate against the NestJS API.
</acceptance_criteria>
</task>
