# Plan 01-02: Core Trip Management & Guest Join

## Objective
Establish the Trip entity, allowing the Leader to create trips and Guests to view the trip itinerary via a shareable link without prior authentication.

## Tasks

<task>
<read_first>
- `packages/database/prisma/schema.prisma`
</read_first>
<action>
Expand Prisma schema to include `Trip` and `TripMember`.
`Trip`: id, name, destination, startDate, endDate, joinCode (String, @unique).
`TripMember`: userId, tripId, role (LEADER, MEMBER).
Run `npx prisma generate`.
</action>
<acceptance_criteria>
- `schema.prisma` contains `model Trip` and `model TripMember`.
</acceptance_criteria>
</task>

<task>
<read_first>
- `apps/api/src/app.module.ts`
</read_first>
<action>
Create a `TripsModule` in NestJS.
Implement `POST /api/trips` for trip creation (Leadership assignment).
Implement `GET /api/trips/:joinCode` (Publicly accessible, no AuthGuard).
</action>
<acceptance_criteria>
- `TripsController` exposes POST and GET routes.
- The default GET route returns trip data securely without leaking private user info.
</acceptance_criteria>
</task>
