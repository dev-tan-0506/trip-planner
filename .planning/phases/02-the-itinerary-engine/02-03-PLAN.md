    2. Register VotesModule in apps/api/src/app.module.ts.
    3. In apps/api/src/votes/votes.controller.ts expose these routes:
       - POST /trips/:tripId/votes/sessions
       - GET /trips/:tripId/votes/sessions/:sessionId
       - POST /vote-sessions/:sessionId/approve
       - POST /vote-sessions/:sessionId/ballot
       - POST /vote-sessions/:sessionId/options
       - POST /vote-sessions/:sessionId/options/:optionId/approve
       - POST /vote-sessions/:sessionId/resolve
    4. In apps/api/src/votes/votes.service.ts implement session lifecycle rules:
       - only trip members can vote
       - members may propose sessions, but non-leader sessions start as PENDING_APPROVAL
       - NEW_OPTION sessions require targetDayIndex and targetInsertAfterItemId
       - REPLACE_ITEM sessions require targetItemId and must expose both the current itinerary item and challenger options
       - ballots update the unique [voteSessionId, userId] row so the latest active vote counts
       - when deadline passes, close the session on the next read/write and compute the winner
       - a tie creates one shorter TIE_BREAK child session; a second tie changes status to LEADER_DECISION_REQUIRED
       - NEW_OPTION winners must persist a VoteSessionOutcome record with winningOptionId, targetDayIndex, targetInsertAfterItemId, and the winning payload snapshot before any follow-up itinerary write; if the approved auto-insert rule runs immediately, that same record must also store createdItemId
       - REPLACE_ITEM winners must persist a VoteSessionOutcome record with winningOptionId, targetItemId, and the winning payload snapshot; if the winner is converted into a leader-review proposal, record replacementProposalId on that same outcome row and never overwrite the itinerary item directly
    5. In apps/api/src/votes/votes.gateway.ts implement room join plus four event names: vote.snapshot, vote.session-updated, vote.results-updated, vote.closed. Every socket join must fetch and emit a fresh snapshot so reconnects recover cleanly.
  </action>
