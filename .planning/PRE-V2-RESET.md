Purpose: reset planning cleanly before opening the next milestone, because the `v2` direction is changing.

## Decide Before Opening v2

1. Confirm whether `v2` is still a web-first extension or a product-direction shift.
2. List what from `v1` is still core and non-negotiable.
3. List what from `v1` should be dropped, rewritten, or deprioritized.
4. Define the new success metric for `v2` in one sentence.
5. Decide whether pending `v1` tech debt should:
   - be promoted into `v2`
   - stay archived as historical debt
   - be explicitly dropped

- Updated product thesis
- New milestone goal
- Any changed user segment or acquisition strategy
- Any changed platform assumptions
- Any new constraints or non-goals
